# Design Rules

## Design Philosophy

The portfolio should look premium, modern and minimal.

The focus should always remain on projects and engineering work rather than unnecessary visual effects.

Animations should improve the experience, not distract from the content.

The website should feel like it belongs to a software engineer.

---

# Theme

Dark Mode Only

No light mode.

Background should remain dark throughout the website.

---

# Responsiveness

Desktop First

Must work perfectly on:

- Desktop
- Laptop
- Tablet
- Mobile

No horizontal scrolling.

All layouts should adapt naturally.

---

# Layout

Maximum content width:
1200px

Consistent spacing between sections.

Large whitespace for readability.

Sections should feel connected.

---

# Cards

Rounded corners.

Subtle shadows only.

No heavy borders.

Hover effects should feel smooth.

---

# Buttons

Primary Button

Filled.

Rounded.

When cursor moves on it the color should change.

Secondary Button

Outline.

Rounded.

Hover should slightly scale.

Transition duration:
250ms–350ms

---

# Typography

Large bold headings.

Readable body text.

Good spacing between paragraphs.

Never use more than two font families.

---

# Icons

Use Lucide Icons or React Icons.

Icons should match the accent color.

Keep icon sizes consistent.

---

# Images

High quality only.

No stretched images.

Profile image should remain the visual focus.

Project screenshots should maintain aspect ratio.

---

# Animations

Smooth.

Subtle.

Professional.

Never distracting.

Avoid excessive bouncing.

---

# Performance

Fast loading.

Lazy load images.

Optimize animations.

Avoid unnecessary JavaScript.

---

# Accessibility

Readable contrast.

Keyboard navigation.

Descriptive button labels.

Semantic HTML.

Alt text for all images.

---

# Code Quality

Reusable React components.

Clean folder structure.

No duplicated code.

Reusable Tailwind classes.

Maintainable architecture.
